# 全国ガチャ旅

ガチャガチャ（カプセルトイ）を回す感覚で、次の**目的地・観光地・ご当地グルメ**をランダムに決められるサイトです。

既存のランダム旅行先決定サイトの多くは「円形ルーレット/スピナー」で演出するのに対し、本サイトは**カプセルトイ機を模した抽選演出**（クランクを回す → カプセルが落下 → バウンド → パカッと割れて結果カードが出現）で差別化しています。

## 特徴

- 🎰 **ガチャガチャ風の抽選演出** — ありがちな円形ルーレットではない、カプセルトイ機のアニメーション
- 📍🗺️🍽️ **3つのカテゴリ** — 目的地 / 観光地 / ご当地グルメを個別に抽選可能
- 🏘️ **目的地は全国約1,700市区町村を収録** — 都道府県単位ではなく市区町村単位で抽選（うち94市区町村は紹介文を個別に執筆、それ以外は都道府県の代表紹介文で補完）
- 🗺️🍽️ **観光地・ご当地グルメは各都道府県6件ずつ（計564件）** — 実在する有名スポット・名物料理を手作業で収録
- 🗾 **地方フィルタ** — 北海道〜九州・沖縄の8地方で候補を絞り込み可能
- 🔁 **連続重複回避** — 同じ結果が続けて出にくいロジック
- 🗺️ **地図を埋め込み表示** — APIキー不要のGoogleマップ埋め込みで結果の場所をその場で確認、大きな地図で開くリンクも用意
- 🐦 **Xでシェア** / 結果をコピー
- ⚙️ **完全静的・ビルド不要** — npmインストールもサーバーサイド処理も不要
- ♿ `prefers-reduced-motion` 設定を尊重し、アニメーションを簡略化

## ローカルでの実行方法

`index.html` をブラウザで直接開くだけで動作します（データはJSファイルに埋め込んでいるため、`file://` で開いてもCORSエラーは発生しません）。

```bash
# 例: 簡易サーバーを立てたい場合（任意）
python3 -m http.server 8000
# → http://localhost:8000 にアクセス
```

## GitHub Pages でのデプロイ

1. GitHubリポジトリの **Settings → Pages** を開く
2. **Source** を「Deploy from a branch」に設定
3. ブランチを選択し、フォルダを `/ (root)` に設定して保存
4. `https://<ユーザー名>.github.io/random_tour/` で公開されます

## データについて

`js/data.js` の `GACHA_DATA` 配列に47都道府県分のデータが入っています。各都道府県は以下の形式です。

```js
{
  id: "hokkaido",
  name: "北海道",
  region: "北海道", // 北海道/東北/関東/中部/近畿/中国/四国/九州・沖縄
  emoji: "🐄",
  destinations: [ { name, emoji, description }, ... ], // 代表的な市区町村
  spots: [ { name, emoji, description }, ... ],
  foods:  [ { name, emoji, description }, ... ],
}
```

目的地カテゴリは都道府県ではなく市区町村単位で抽選されます（全国合計1,739件）。主要な94市区町村は個別に紹介文（description）を執筆済みですが、それ以外の市区町村は「〇〇県にある市区町村。」という共通の紹介文で補完しています。観光地・ご当地グルメは各都道府県6件ずつ（全国合計282件×2カテゴリ）、いずれも実在するスポット・料理を個別の紹介文つきで収録しています。目的地・観光地・ご当地グルメを追加・編集したい場合は、該当都道府県の `destinations` / `spots` / `foods` 配列に項目を追加してください。

## データ・免責について

掲載している市区町村名・観光地・ご当地グルメは代表的な一例です。市区町村の一覧は複数の情報源をもとに作成していますが、平成の大合併以降の名称変更等により、まれに誤りや抜け漏れが含まれる可能性があります。最新情報や正確な行政区分は各都道府県・市区町村の公式サイトでご確認ください。誤りにお気づきの場合は `js/data.js` を修正してください。
